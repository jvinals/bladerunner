## Learned User Preferences
- When generating Playwright for modern SPAs, bake strict anti-flakiness interaction rules into the prompt/template instead of relying on generic selectors or `.fill()` for custom controls.
- For LLM-driven browser actions, include a tagged Set-of-Marks screenshot plus a matching accessibility or actionable-node tree in the model context.
- For AI prompt and vision flows, phrase steps as neutral QA on staging or local apps (e.g. verify errors, permissions, expected messages) rather than attack-like imperatives, to reduce empty model output from safety refusals.

## Learned Workspace Facts
- Autonomous **evaluations**: use **Start run** only when status is **QUEUED**; after **FAILED**, **COMPLETED**, **CANCELLED**, or **WAITING_FOR_HUMAN**, use **Retry run** / **Re-run** (reprocess), which resets artifacts and re-queues. A second **Start run** while a run is already in progress returns `scheduled: false` and does not start a parallel orchestrator loop.
- BladeRunner's default Clerk OTP mode for automatic sign-in should be MailSlurp when no explicit mode is selected.
- Automatic sign-in during recording and playback is modeled as one first-class step, with provider-specific auth metadata kept separate from the normal Playwright step chain.
- Generic email/password automatic sign-in should tolerate delayed auth UI and staged email-then-password flows (and similar) before failing; OTP completion must not treat same-host URLs as success while the OTP UI is still active.
- For autonomous **evaluations**, when **auto sign-in** is enabled for the run, the evaluation **codegen** and **analyzer** LLM prompts should include that fact so the models do not choose **ask human** for credentials or emit Playwright that types secrets; the orchestrator runs automated sign-in before each step when configured.
- **AI Prompt** / **AI Visual ID**: split script generation from script execution; `Done` should only succeed after both have run and should not rerun a test that already passed. AI Visual ID is run-scoped. In the Add AI prompt step drawer, keep the `5.- AI Visual ID` block below item #4 (not above the numbered steps). The Tree modal should show a Playwright-style accessibility snapshot (role, accessible name, and key attributes in a clear hierarchy), not an unstructured tree dominated by unnamed nodes.
- For resumable runs, run detail should key playback and continue-recording off whether a **live** recording session exists (not only persisted `RECORDING` status); keep `Continue recording` available during playback and do not block handoff to the recording workspace on awaiting full playback teardown (stop playback in the background).
- **Project discovery** (Run app discovery): live JPEG frames use the recording socket channel id **`discovery-{projectId}`** (join `run:discovery-{projectId}`); exploration uses **LLM-driven Playwright steps**. The model must not set **stop** until **both** a minimum number of **executed steps** and a minimum count of **distinct normalized URLs** are met (unless truly blocked), in addition to global step/wall caps. Before each LLM capture, the browser **scrolls** the main document and common overflow containers; **visited screens** and the **navigation Mermaid** map use **logical screens** (normalized URL + normalized title) so SPAs without URL changes still add nodes. Mermaid node labels include **pathname** when useful to avoid duplicate titles (e.g. two “Dashboard” views). In-progress discovery can be **cancelled** (`POST /projects/:id/discovery/cancel` or **Cancel discovery** on Projects); the run ends **failed** with a cancellation message so the user can start again. When an explore **Playwright** snippet **fails**, the next LLM turn must include the **failed code + error** so the agent does not repeat the same locator; **playback** should expand **button**-labeled select triggers (e.g. “Select Provider”) with combobox-style fallbacks; **modal** flows should scope locators under **`getByRole('dialog', …)`** when the snapshot shows a dialog.
- **Playwright / browser-worker**: playback must rewrite or escape locator strings that embed single quotes inside `.locator('...')` / `.locator("...")` (e.g. `[class*='...']`) so compiled snippets stay valid JavaScript. The **browser-worker** should pass `channel: 'chromium'` into `chromium.launchServer` (especially when headless) so Playwright uses the main Chromium bundle from `playwright install chromium`, not the separate `chromium-headless-shell` path that often triggers missing-executable errors locally. On **Shadcn / Radix** UIs, a labeled row is not always `role="combobox"`—**Radix Select** triggers are often `role="button"`; if `getByRole('combobox', …)` times out, locate the field trigger as a **button** (or by placeholder / visible label) and keep interactions atomic with portaled listbox/search inputs as in the shared codegen guidelines.
- **Evaluations API / timeline**: `POST /evaluations/:id/start` does not reset a FAILED evaluation to a runnable state; Retry uses `POST .../reprocess` (`resetForReprocess` → QUEUED). Calling Start while status is FAILED yields "Evaluation is not runnable". Emit early `evaluationProgress` (`proposing` with sequence) before slow per-step work (auto sign-in, SOM capture); replay the latest progress payload when the client joins the run room so the first event is not dropped. Live step cards should derive codegen/analyzer loading from persisted step JSON (treat stored values including `{}` as present), not only `phase === 'proposing'`, so spinners clear after refetch. Playwright thrown from generated code via eval often stacks as `eval (<anonymous>:...)` only; failure classification must still treat these as recording automation paths so unhandled rejections do not crash the API process.
- **Prisma / migrations**: the schema lives under **`apps/api/prisma/`**; run **`pnpm run migrate:api`** from the repo root (or run `prisma migrate deploy` inside **`@bladerunner/api`**). Running `prisma migrate` from the repo root without `--schema` fails because the default schema path is not there.
- **LLM JSON in the API**: `parseJsonFromLlmText` must tolerate markdown code fences labeled `json` (including a **missing closing** fence), natural-language prose before/after the object, and **CoT** mixed with JSON; use fence stripping plus **balanced-brace** extraction of the first top-level `{ ... }` (string-aware), not only `JSON.parse` on the full raw string.
