## Learned User Preferences

- On evaluation run **Thinking process** (sidebar panel): **accordion**—at most one row expanded; expanding a row collapses the previously expanded one. Auto-expand the **active** codegen/analyzer step when it becomes active. Do not strikethrough completed substeps.
- When generating Playwright for modern SPAs, bake strict anti-flakiness interaction rules into the prompt/template instead of relying on generic selectors or `.fill()` for custom controls.
- For LLM-driven browser actions, include a tagged Set-of-Marks screenshot plus a matching accessibility or actionable-node tree in the model context.
- For AI prompt and vision flows, phrase steps as neutral QA on staging or local apps (e.g. verify errors, permissions, expected messages) rather than attack-like imperatives, to reduce empty model output from safety refusals.

## Learned Workspace Facts

- **Evaluations (lifecycle, API, orchestrator, codegen)**: use **Start run** only when status is **QUEUED**; after **FAILED**, **COMPLETED**, **CANCELLED**, or **WAITING_FOR_HUMAN**, use **Retry run** / **Re-run** (`resetForReprocess` → QUEUED). A second **Start run** while in progress returns `scheduled: false`. `POST /evaluations/:id/start` does not reset FAILED to runnable. **Automatic** runs advance without a **Continue** prompt each step. **Single vision LLM per iteration** (`evaluation_codegen`): JSON `**stepMode`** is `**execute_playwright**` (then run Playwright + deterministic retry/advance), `**finish**` (end run, no Playwright), or `**ask_human**` (human question, no Playwright; coerced to **retry** while automatic sign-in still running). No `**evaluation_analyzer`** call. Emit early `evaluationProgress` and replay it when the client joins. Live step cards derive codegen/analyzer loading from persisted step JSON (including `{}`), not only `phase === 'proposing'`. **evaluation_codegen** `**AbortSignal.timeout`**: on timeout/abort persist `**codegenTimedOut**`, empty Playwright, **continue** the run. `**isAbortOrTimeoutError`** must handle `**DOMException**`. Eval `**eval(...)**` Playwright errors may stack as `eval (<anonymous>:...)`; still classify as recording automation so the API does not crash.
- **Automatic sign-in**: default Clerk OTP mode **MailSlurp** when unset. One first-class step; provider metadata stays off the normal Playwright chain. Tolerate delayed/staged auth UI before failing; do not treat same-host navigation as OTP success while the OTP UI is active. When **auto sign-in** is on, **codegen** prompts must say so—avoid **ask human** for credentials or Playwright that types secrets; the orchestrator runs sign-in before steps.
- **AI Prompt** / **AI Visual ID**: split script generation from script execution; `Done` should only succeed after both have run and should not rerun a test that already passed. AI Visual ID is run-scoped. In the Add AI prompt step drawer, keep the `5.- AI Visual ID` block below item #4 (not above the numbered steps). The Tree modal should show a Playwright-style accessibility snapshot (role, accessible name, and key attributes in a clear hierarchy), not an unstructured tree dominated by unnamed nodes.
- For resumable runs, run detail should key playback and continue-recording off whether a **live** recording session exists (not only persisted `RECORDING` status); keep `Continue recording` available during playback and do not block handoff to the recording workspace on awaiting full playback teardown (stop playback in the background).
- **Project discovery** (Run app discovery): live JPEG frames use the recording socket channel id `**discovery-{projectId}`** (join `run:discovery-{projectId}`); exploration uses **LLM-driven Playwright steps**. The model must not set **stop** until **both** a minimum number of **executed steps** and a minimum count of **distinct normalized URLs** are met (unless truly blocked), in addition to global step/wall caps. Before each LLM capture, the browser **scrolls** the main document and common overflow containers; **visited screens** and the **navigation Mermaid** map use **logical screens** (normalized URL + normalized title) so SPAs without URL changes still add nodes. Mermaid node labels include **pathname** when useful to avoid duplicate titles (e.g. two “Dashboard” views). In-progress discovery can be **cancelled** (`POST /projects/:id/discovery/cancel` or **Cancel discovery** on Projects); the run ends **failed** with a cancellation message so the user can start again. When an explore **Playwright** snippet **fails**, the next LLM turn must include the **failed code + error** so the agent does not repeat the same locator; **playback** should expand **button**-labeled select triggers (e.g. “Select Provider”) with combobox-style fallbacks; **modal** flows should scope locators under `**getByRole('dialog', …)`** when the snapshot shows a dialog.
- **browser-worker WebSocket**: transient Fly/proxy errors (**`ECONNRESET`**, etc.) are handled with **retries + backoff** on the control **`ws://`** connection. The worker listens on an explicit **`0.0.0.0`** HTTP server for Flycast; **`min_machines_running = 1`** on Fly avoids cold-start races when possible.
- **Fly.io API listen address**: Nest must **`listen` on `API_HOST`** (default **`0.0.0.0`**) so Fly deploy health checks to **`0.0.0.0:<internal_port>`** succeed; **`app.listen(port)`** without a host can cause **`[PM05] failed to connect to machine`**.
- **`BROWSER_WORKER_URL` normalization**: On a **Fly Machine** (`FLY_ALLOC_ID`), legacy **`appname.internal`** is rewritten to **`appname.flycast`** (log warning) so worker DNS works when Machines scale to zero. **Locally** (no `FLY_ALLOC_ID`), **`*.internal`** / **`*.flycast`** are coerced to **`ws://127.0.0.1:3002`**. Remove stale Fly URLs from **`.env`** or Fly **secrets** if you still see **`ENOTFOUND`** after deploy.
- **Playwright / browser-worker**: playback must rewrite or escape locator strings that embed single quotes inside `.locator('...')` / `.locator("...")` (e.g. `[class*='...']`) so compiled snippets stay valid JavaScript. The **browser-worker** should pass `channel: 'chromium'` into `chromium.launchServer` (especially when headless) so Playwright uses the main Chromium bundle from `playwright install chromium`, not the separate `chromium-headless-shell` path that often triggers missing-executable errors locally. Avoid `**host: '::'`** for `**launchServer**` (can yield invalid CDP URLs like `**ws://:::port/...**`); bind `**0.0.0.0**` and when `**WORKER_EXTERNAL_HOST**` is unset **normalize** the returned `**wsEndpoint`** to `**127.0.0.1**` so the API’s `**chromium.connect**` gets a valid URL. On **Shadcn / Radix** UIs, a labeled row is not always `role="combobox"`—**Radix Select** triggers are often `role="button"`; if `getByRole('combobox', …)` times out, locate the field trigger as a **button** (or by placeholder / visible label) and keep interactions atomic with portaled listbox/search inputs as in the shared codegen guidelines. Do not use long **utility-class** chains as raw CSS in `.locator()`—they are often invalid selectors; prefer **getByRole**, **getByLabel**, **getByTestId**, or other stable locators.
- **Evaluation LLM prompts & capture**: Omit project/run **discovery** instructions from codegen, analyzer, and agent prompts unless the step explicitly needs them. Before capturing viewport JPEG + SOM/a11y for those models, **settle** the page (wait for load and meaningful interactive UI) so previews and prompts match; blank JPEGs with empty SOM/a11y usually mean capture ran before the SPA finished rendering. **Shell-first SPAs** often paint **sidebar/header** before the `**main`** canvas fills in — persisted previews can look “partially formed” compared to the **live screencast** if capture runs when only chrome exists; vision settle should treat **primary content** (e.g. `main` / `[role="main"]`) and **visual density**, not only global interactives, so the stored JPEG tracks what operators see in the stream.
- **LLM JSON & OpenRouter**: `parseJsonFromLlmText` must tolerate markdown code fences labeled `json` (including a **missing closing** fence), natural-language prose before/after the object, and **CoT** mixed with JSON; use fence stripping plus **balanced-brace** extraction of the first top-level `{ ... }` (string-aware), not only `JSON.parse` on the full raw string. `**chatJson`** and OpenAI-compatible `**OpenAiProvider**` / `**OpenRouterProvider**` default to `**responseFormat: 'text'**` (not API `**response_format: json_object**`); obtain JSON via prompts plus `**parseJsonFromLlmText**` (opt-in `**json_object**` only for strict OpenAI-only callers). For **OpenRouter multimodal** requests, send `**provider: { ignore: ['amazon-bedrock'] }`** when needed—Bedrock may route Claude Haiku/Sonnet snapshots that **reject image input**, surfacing as generic **400** “**Provider returned error**” with a nested upstream message.
- **Deployment (Vercel, Fly.io, Railway)**: Frontend (`apps/web`) → **Vercel**; API + **browser-worker** → **Fly.io** same region + **Flycast** (CDP is chatty—co-locate). PostgreSQL → **Railway**. Browser-worker is **internal-only**. `**BROWSER_WORKER_URL`** / `**WORKER_EXTERNAL_HOST**` must use the worker’s `***.flycast**` hostname, not `**.internal**`, when Machines can scale to zero—stopped Machines do not get `**.internal**` DNS (`**ENOTFOUND**`). Ensure a private Flycast IP exists on the worker (`**fly ips allocate-v6 --private**`) if needed. **`apps/browser-worker`** Docker/Fly image builds from **repo root** with **`pnpm install --filter @bladerunner/browser-worker...`** and **`pnpm-lock.yaml`** (same idea as **`@bladerunner/api`** — no per-app **`npm`** lockfile). GitHub Actions **`fly-deploy-api.yml`** and **`fly-deploy-browser-worker.yml`** deploy to Fly on **`main`** (path-filtered); **Vercel** production/preview is usually the dashboard **Git** integration, not a workflow file in this repo. Each app has its own `fly.toml` / `Dockerfile.production`; API: `**fly deploy --config apps/api/fly.toml**` from **repo root**; `.dockerignore` excludes `node_modules`; Playwright Docker image tag matches `**playwright-core`**. `**vercel.json**` at **repo root** with `**"framework": null`**; build `pnpm --filter @bladerunner/web exec vite build`; output `apps/web/dist`; dashboard overrides beat `vercel.json`—clear them to use the file.
- **Prisma / migrations**: schema under `**apps/api/prisma/`**; `**pnpm run migrate:api**` from repo root (or `prisma migrate deploy` in `**@bladerunner/api**`). Root `prisma migrate` without `**--schema**` fails.
